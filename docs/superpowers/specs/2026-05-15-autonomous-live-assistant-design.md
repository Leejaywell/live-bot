# 2026-05-15 自主场控 AI 智能体 (Streamix Pro) 设计方案

## 1. 愿景与目标
打造一款具备“全景感知、深度记忆、自主行动”能力的商业化直播助手。通过混合认知架构，降低主播运营负担，提升观众粘性。

## 2. 核心架构：混合认知模型 (Hybrid Cognitive Architecture)
采用“本地边缘计算 + 云端大脑”的协作模式。

- **本地端 (Local Edge)**: 
  - 负责 ASR 识别、弹幕监听、OBS 实时控制。
  - 执行视觉预过滤（帧差检测），仅将关键变化帧（快照）上传云端。
  - 维护临时上下文，确保 100ms 内的指令级响应。
- **云端 (Cloud Brain)**:
  - 负责多模态视觉分析（LLM + Vision）。
  - 托管长期记忆系统（观众画像知识图谱）。
  - 提供高情感表现力的 TTS 合成引擎。

## 3. 功能设计

### 3.1 视觉感知系统 (Vision System)
- **采集方式**: 截屏模式（支持全屏或应用窗口）。
- **识别类别**:
    1. **游戏事件**: 胜负、结算、金光抽取、关键数值。
    2. **主播状态**: AFK（离开）、饮水、状态低迷感知。
    3. **氛围感知**: 弹幕视觉特效识别、礼物连击识别。
    4. **自定义监控**: 支持主播手动框选识别区域。

### 3.2 长期记忆与情感连接 (Long-term Memory)
- **数据结构**: 结构化观众档案 (JSONB) + 向量化历史片段。
- **内容维度**:
    - **生活点滴**: 观众提到的职业、考试、宠物、情感状态等细节。
    - **互动偏好**: 喜欢的音色、常玩的梗、性格色彩（调皮/严谨）。
- **同步机制**: 云端托管，支持主播在不同设备登录时记忆无缝同步。

### 3.3 自主 OBS 自动化 (Autonomous Control)
- **模式**: 共治模式 (Co-governance)。
- **典型场景**:
    - 主播去洗手间 -> AI 自动切到“休息中”场景，并开始与弹幕玩小游戏。
    - 识别到游戏高光 -> 自动触发 OBS 特效，并在直播间发送弹幕标记。
    - 检测到人气下滑 -> 弹出悬浮建议提醒主播互动或更换话题。

## 4. 技术栈选择
- **前端**: React (TypeScript) + Tauri (用于控制本地 OBS 和截屏)。
- **后端**: Rust (核心调度、本地模型推理、SQLite 存储)。
- **云端服务**: 
    - 视觉/逻辑: GPT-4o / Claude 3.5 Sonnet / DeepSeek Vision。
    - 记忆系统: Postgres + pgvector / Redis。
    - 语音合成: Bert-VITS2 (集成在云端)。

## 5. 开发路线图 (Roadmap)
1. **Phase 1 (Visual OBS)**: 实现截屏预处理流程与 OBS WebSocket 控制闭环。
2. **Phase 2 (Memory)**: 搭建云端记忆同步系统与观众画像提取逻辑。
3. **Phase 3 (Emotion)**: 整合视觉与记忆，实现具备人格化特征的互动引擎。

## 6. 商业化考量
- **成本控制**: 采用本地过滤技术，将单场直播的云端 Token 消耗控制在可盈利范围内。
- **隐私保护**: 所有的视频流处理均在内存中完成，快照上传需经过脱敏处理（如模糊人脸）。
