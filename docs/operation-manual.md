# 流光操作手册

更新日期：2026-05-20

## 1. 本地开发

### 1.1 环境准备

- Rust stable
- Node.js 20+
- npm
- macOS 需要 Xcode Command Line Tools
- Windows 需要 Visual Studio Build Tools / MSVC

### 1.2 安装前端依赖

```bash
cd src-tauri
npm install
cd ..
```

### 1.3 启动桌面开发模式

```bash
cargo run --features tauri
```

### 1.4 常用检查

```bash
cargo check --features tauri
cargo check -p streamix-voice --features vad,voice-changer,local-tts
cd src-tauri && npm run build
```

## 2. 首次使用

1. 启动应用。
2. 打开登录弹窗，使用 Bilibili App 扫码。
3. 登录成功后选择直播间或使用账号绑定的直播间。
4. 确认房间状态后启动监听。
5. 在实时弹幕页确认事件流是否出现。
6. 需要 OBS 展示时，进入插件中心复制对应 URL 到 OBS 浏览器源。

## 3. 常用直播操作

### 3.1 启动监听

- 登录必须有效。
- 需要先连接房间。
- 实时弹幕页和直播概览页都可以查看监听状态。

### 3.2 发送弹幕

- 在实时弹幕页输入内容后发送。
- 后端通过当前登录态调用 Bilibili API。
- 发送失败通常与登录失效、风控、频率限制或网络有关。

### 3.3 配置自动回复

进入「触发回复」：

- 欢迎语：开启通用欢迎语，维护模板；需要指定用户时添加 UID 欢迎语。
- 粉丝互动：开启关注/分享答谢，维护附加模板。
- 礼物感谢：开启礼物、SC、聚合、最低金额、别名和模板。
- 定时任务：添加 cron 和弹幕内容。
- 过滤规则：添加 UID 黑名单、昵称关键词、敏感词和重复阈值。

保存后，如果监听正在运行，后端会接收配置更新命令。

### 3.4 使用 AI 助手

1. 在「智能助手」添加 LLM Provider。
2. 添加或启用 AI Bot，并绑定 Provider。
3. 配置 Bot 昵称、提示词和启用状态。
4. 直播间观众通过 `@昵称 内容`、包含昵称或裸 `@内容` 触发。
5. 如需语音播报，启用 TTS 并配置 TTS Provider。

注意：AI 页面测试对话用于快速检查配置，不完全等同于直播 AgentRuntime 路径。

### 3.5 使用语音陪伴

1. 在「模型管理」检查或下载需要的本地模型。
2. 在「语音陪伴」配置 VAD、ASR、TTS、音色、语速、音调和麦克风增益。
3. 开启语音能力后，应用会在 TTS 播放时临时关闭麦克风输入，避免回声。
4. 如果识别或播放异常，先检查模型状态、Provider 配置和系统音频设备。

### 3.6 使用音乐互动

1. 进入「插件中心 / 音乐互动」。
2. 开启音乐互动。
3. 配置播放方式、OBS 展示样式和礼物档位。
4. 复制 OBS URL 到浏览器源。
5. 观众送礼或 SC 获得点歌权益。
6. 观众发送 `点歌 歌名 歌手` 搜索。
7. 观众发送 `确认 1` 或 `点歌 #1` 消耗权益并加入队列。
8. 主播在队列中打开、完成、跳过或标记失败。

## 4. OBS 插件

插件 URL 在对应插件页复制：

- 弹幕聊天
- 音乐互动
- 心愿目标
- 抽奖互动
- 礼物特效
- 最近礼物
- 礼物排行

OBS 浏览器源建议：

- 宽高按插件页面配置设置。
- 需要透明背景时启用插件透明选项。
- 如果页面没有刷新，先确认本地插件服务端口没有被占用。

## 5. 本地文件位置

应用运行数据不在仓库目录内，默认位置由系统决定。

- 主配置：`streamix.toml`
- 插件配置：`plugin-settings.toml`
- 数据库：`streamix.db`
- 登录态：token/session 文件
- 模型：应用数据或资源目录下的模型目录

在应用内可通过「打开配置目录」快速定位。

## 6. 常见问题

### 登录失效

重新扫码登录。若前端定期检查发现登录失效，会弹出登录窗口。

### 启动监听失败

检查登录态、房间号、网络和 Bilibili 直播间状态。必要时停止监听后重新连接房间。

### OBS 页面空白

检查插件 URL 是否为 `127.0.0.1` 本地地址，确认应用仍在运行，确认端口未被占用。

### 本地模型不可用

进入模型管理重新检查模型状态。下载失败时可取消后重试，并确认网络和磁盘空间。

### TTS 没有声音

检查 TTS Provider、音色、系统输出设备和音量。直播监听中的 TTS 配置变更后可使用重载语音/TTS 相关操作刷新运行时。
