# 流光软件功能逻辑文档

更新日期：2026-05-20

## 1. 主流程

```text
启动应用
  → 检查登录态
  → 选择或恢复直播间
  → 启动监听
  → 接收直播事件
  → 记录数据 + 更新插件 + 触发规则/AI/语音
  → 前端和 OBS 页面实时展示
```

## 2. 登录与房间逻辑

1. 前端调用 `start_login` 获取二维码。
2. 前端每 2 秒调用 `poll_login`。
3. 登录成功后写入 session，并刷新用户信息。
4. 如果存在上次连接房间，前端调用 `check_room` 并尝试 `start_monitor`。
5. 连接房间后保存 `connected_room`，用于下次恢复。

## 3. 事件处理链

直播事件进入后端后按以下逻辑处理：

1. 解析为 `LiveEvent`，保留原始 JSON。
2. 写入 `interaction_records`，并更新用户统计。
3. 推送 `live-event` 给前端。
4. 根据事件更新插件配置状态：心愿目标、抽奖、礼物特效、最近礼物、礼物排行。
5. 缓存礼物目录信息。
6. 执行 BotEngine 规则：黑名单、过滤、欢迎、帮助、答谢、PK/活动提示。
7. 如果音乐互动开启，礼物/SC 生成点歌权益，弹幕指令触发搜索/确认/查询/取消。
8. 如果 AI Bot 命中触发条件，进入 AgentRuntime。
9. 需要播报时进入 SpeakerRouter。

## 4. 自动回复逻辑

### 4.1 欢迎语

- 进场事件先匹配 `SpecialWelcomeList`，按 UID 精准匹配。
- 命中特定欢迎语后直接返回，不再走通用欢迎语。
- 未命中时，如果 `GeneralWelcomeEnabled` 开启，从 `GeneralWelcomeMsgs` 随机选择。

### 4.2 过滤与黑名单

- 永久黑名单不受 `DanmuFilterEnable` 控制。
- UID 命中 `PermanentBlacklistUsers` 或昵称包含 `PermanentBlacklistNames` 时，事件不再触发回复。
- 弹幕过滤开启后，文本包含 `DanmuFilterWords` 或同一用户重复同一文本达到阈值时丢弃。

### 4.3 礼物与关注答谢

- 关注和互相关注受 `ThanksFocus` 控制。
- 分享受 `ThanksShare` 控制。
- 舰长购买和普通礼物答谢受 `ThanksGift` 相关配置影响。
- SC 答谢受 `ThanksSuperChat` 控制。
- 礼物聚合按超时时间合并同一用户礼物，支持最低金额过滤、别名和模板。

### 4.4 定时弹幕

- `CronDanmu` 开启后，每条 `CronDanmuList` 创建独立任务。
- 5 位 cron 会补秒转换为 6 位表达式。
- 消息可以随机选择，也可以按顺序轮换。

## 5. AI 逻辑

直播弹幕触发顺序：

1. 精确 `@{bot.nickname}` 前缀。
2. 文本包含 bot 昵称。
3. 裸 `@` 前缀时使用第一个启用 Bot。

命中后：

1. 提取 prompt。
2. 根据 Bot 绑定 Provider 发起 LLM 请求。
3. AgentRuntime 最多执行多轮工具调用。
4. 结果写入会话记忆。
5. 回复弹幕格式为 `[{nickname}]{reply}`。
6. 如果 TTS 开启，使用 AI 播报通道播放。

## 6. 音乐互动逻辑

### 6.1 权益产生

- 礼物或 SC 事件按金额匹配启用档位。
- 达到多个档位时取门槛最高的档位。
- 权益绑定直播场次、房间、UID 和用户昵称。
- 已使用或过期权益不能再次消费。

### 6.2 点歌指令

支持的观众指令：

- `点歌 <关键词>`：搜索歌曲。
- `点歌 #<编号>`、`点歌 #<编号> 确认`、`确认 <编号>`、`选 <编号>`：确认候选。
- `我的点歌`：查询自己的点歌状态。
- `我的积分`、`点歌积分`、`我的档位`、`点歌档位`：查询权益。
- `取消点歌`：取消自己的未播放点歌。

### 6.3 搜索与确认

- 当前已接入网易云 Provider。
- 播放方式选择网易云时只允许网易云候选。
- 选择 QQ 音乐时，如果 Provider 未接入，返回明确提示。
- 搜索候选写入 `song_search_contexts`，确认时必须使用最近未过期候选。
- 确认成功后写入 `song_requests`，消费一条权益。

### 6.4 队列状态

- `queued`：等待播放。
- `playing`：正在播放，同场次同房间只能有一个。
- `finished`、`skipped`、`failed`：终态。
- 优先级由档位基础分、礼物金额和重复惩罚共同决定。

## 7. 插件状态逻辑

- 插件配置集中保存在 `plugin-settings.toml`。
- 直播事件会实时更新需要持久化的插件状态。
- 更新成功后广播配置变更，OBS 页面通过接口或 WebSocket 刷新。
- 插件页面提供模拟按钮，用于不依赖真实直播事件调试画面。

## 8. 数据统计逻辑

- 互动记录按场次、房间、用户和时间维度存储。
- 数据统计页按周期读取汇总。
- 观众档案读取 `tracked_users` 和画像统计，支持人工编辑别名、备注和专属音色。
- 数据清理循环会删除过旧的交互记录，避免本地库无限增长。
