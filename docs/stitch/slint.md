# Live Bot 直播弹幕机器人

## §0 · 项目设定（首次发给 Stitch）

```
产品：Live Bot 直播弹幕机器人 — Slint 原生桌面端
平台：Slint 1.16，设计基准 1080×700，最小窗口 920×580
界面语言：简体中文，所有标签、按钮、提示、占位符均为中文。等宽字体仅用于 房号 / UID / 时间戳 / 日志 / cron / JSON
字体：PingFang SC / SF Pro（中文）+ SF Mono（等宽），基础字号 12px

视觉语言：macOS Big Sur 之后的 Liquid Glass —— 半透明白卡 + 漂浮高斯色团 + 镜面高光

【背景】整窗 160deg 线性渐变：
- 浅色：linear-gradient(160deg, #eaeef5, #dfe3eb)
- 深色：linear-gradient(160deg, #14141b, #0c0c10)

背景上漂浮 3 个 460-520px 高斯色团（radial-gradient + 高斯模糊 80），所有页面共享：
- 左上 主色团（浅色透明度 70% / 深色 78%）位置 x=-160 y=-180
- 右上 #ff2d55 玫红团（浅 82% / 深 85%）
- 右下 #34c759 翠绿团（浅 83% / 深 86%）

【布局】2 区，无右侧面板：
- Topbar（高 52）：玻璃带 #ffffff80→#ffffff50（浅）/ #ffffff0e→#ffffff04（深）
- Sidebar（宽 196，可折叠）：玻璃 #ffffff7a→#ffffff48（浅）/ #ffffff10→#ffffff05（深），右侧 1px 细分隔
- Main：可滚动，padding 14；所有卡片圆角 14-18，drop-shadow 22 blur 6 y #0000001c

侧边栏 7 项单段：仪表盘 / 登录与房间 / 监听与发送 / 自动回复 / AI 机器人 / 数据统计 / PK 与活动 / 系统与更新

NavItem（高 44 圆角 12）：
- 默认：透明背景，icon 28×28 玻璃方块（白渐变 + 1px rim）
- 激活：accent 半透明渐变（透明度 0.84-0.92）+ accent 边框（透明度 0.7）+ icon 方块换为 accent 实色 + 白图标
- hover：白色微提亮 #ffffff70-#ffffff40

主题：仅 浅色 / 深色 两态（无"跟随系统"，桌面端简化）
主色 4 个预设（按 theme-choice 切换）：
- iOS 蓝 #007aff（默认 0）
- 翠绿 #34c759（1）
- 橙 #ff9500（2）
- 玫红 #ff2d55（3）

自定义颜色选择器（紧凑版，无色环节省空间）：
- 三滑块 H/S/L 每行 36 高，右侧数字输入
- HEX 输入 + 复制
- 实时预览 3 个 60×30 小方块（按钮 / 激活 NavItem / Toggle 开态）
- 收藏 8 格

【液态玻璃组件规范】
- Btn 胶囊（高 30 圆角 15）：
  · 主按钮 primary：linear-gradient(180deg, accent.brighter 10%, accent.darker 4%) + accent.darker 18% 边框 + 10px accent 透明阴影
  · 默认 default：linear-gradient(180deg, #ffffffd6, #ffffff8a) + rim + 4px 阴影
  · 顶部 1px 镜面高光（x:5, w:parent-10, h:1, bg #ffffffe6 / primary 时 #ffffff60）
  · hover：阴影模糊增至 7px / 16px
- IconBtn 圆形 30×30：半透明白渐变，激活时 accent 半透明背景
- Toggle（36×20 胶囊）：
  · 关：linear-gradient #d4d4d9.darker→#d4d4d9 + rim
  · 开：linear-gradient accent.brighter 8%→accent + 8px accent 阴影
  · 滑块 16×16 圆 白渐变 + 4px 黑阴影；位置 x 2 ↔ 17，160ms ease-in-out
- AutoCard（自动化功能卡，14 圆角）：
  · 半透明白卡 + 1px rim + 14px 阴影
  · 左上 28×28 圆角方块图标（accent tint 径向渐变）
  · 右上 Toggle
  · 标题 12/600 + 描述 10/wrap

字号档：标题 17/700 / 区段 11/700 letter-spacing 0.6 / 正文 12/500 / 副文 10/600 letter-spacing 0.4 / 等宽 SF Mono 11/500

设计语言要点：
- 卡片永远半透明 — 透出底色团
- 圆角更大（14-22）字号更小（10-13）
- 没有密集表格 — 全部用卡片网格 + chip 列表
- 没有右侧面板 — 实时状态、日志、用户查询都收进 Main 内的卡片
- 间距舒展，padding 12-14

业务术语：弹幕 / 礼物 / 进场 / 关注 / 分享 / 大航海 / 红包 / PK / 天选 / 粉丝牌 / 舰长（提督 / 总督）/ 财富等级 / 房号 / UID / 电池
样例数据：房号 8792912、主播"花花直播姬"、用户"大老板/小白同学/小迷妹/路人甲"、粉丝牌"花花团 21"、礼物"辣条/小花花/牌子/嘉年华/总督"
```

## 使用建议

- 顺序：§0 → S1（仪表盘验证玻璃感）→ S9（主题面板）→ S2..S8
- 第一段一定要让 Stitch 把"漂浮高斯色团"画出来（左上蓝 / 右上玫红 / 右下翠绿），后面所有页面都要复用同一套色团。如果首图玻璃感不到位，反复调整 §0 中"色团透明度"和"卡片半透明度"两个参数
- 不要让 Stitch 在 Slint 版里画密集表格 — 不符合视觉语言。如果它生成了表格，就用"换成 chip 列表"指令矫正
- Slint 版只做 浅色 / 深色 两态，不需要"跟随系统"画面
