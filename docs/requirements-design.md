# 流光需求设计文档

更新日期：2026-05-20

## 1. 产品定位

流光（Streamix）是面向 Bilibili 直播间的桌面场控系统。当前版本以 Rust/Tauri 桌面端为主，整合直播监听、弹幕发送、自动化回复、AI 助手、语音陪伴、观众档案、数据统计和 OBS 浏览器源插件。

产品目标不是单点的自动回复工具，而是把直播间事件、主播配置、AI/语音能力、插件画面和本地数据沉淀放在同一个工作台中管理。

## 2. 核心用户

- 主播：需要开播前快速登录、连接房间、启动监听和控制互动功能。
- 场控/运营：需要维护欢迎语、礼物答谢、过滤规则、观众档案和数据复盘。
- 直播画面搭建者：需要复制 OBS 浏览器源 URL，调试弹幕聊天、点歌、心愿目标、抽奖、礼物特效等插件。

## 3. 当前需求范围

### 3.1 登录与房间

- 支持 Bilibili 二维码登录、轮询登录状态、登录态持久化。
- 支持读取当前账号信息、按 UID 获取直播间、检查房间状态。
- 支持恢复上次连接房间，并在登录有效时尝试自动恢复监听。
- 支持开播、下播、更新直播间信息、获取分区和推流地址等主播操作。

### 3.2 直播监听与发送

- 连接 Bilibili 直播 WebSocket，接收弹幕、进场、关注、分享、礼物、舰长、SC、红包、PK、禁言、人气等事件。
- 事件进入统一处理链：记录数据、更新插件状态、触发规则、触发音乐互动、触发 AI 和 TTS。
- 支持手动发送弹幕，实时弹幕页展示监听日志并提供暂停、过滤和发送入口。

### 3.3 自动化互动

- 通用欢迎语：观众进场时随机发送模板，支持 `{user}`。
- 特定欢迎语：按 UID 精确匹配，优先级高于通用欢迎语。
- 关注/分享答谢：按开关发送感谢语，可追加关注模板。
- 礼物答谢：支持礼物聚合、最低金额、礼物别名、自定义模板、汇总感谢和盲盒盈亏统计。
- SC 答谢：通过独立开关控制。
- PK/红包/禁言提示：按配置开关输出提示。
- 定时弹幕：支持 cron 表达式和消息轮换/随机发送。
- 弹幕过滤：支持永久黑名单、昵称关键词、敏感词和重复弹幕阈值。

### 3.4 AI 与语音

- AI Provider 按 `llm`、`asr`、`tts` 分类配置。
- AI Bot 与 Provider 分离，直播弹幕触发时使用 Bot 昵称、裸 `@` 或包含昵称等方式进入回复链。
- AI 直播回复使用 AgentRuntime，支持会话记忆和工具调用。
- 已内置工具包括发送弹幕和查询本场统计。
- TTS 支持 Edge、MiniMax、火山引擎、Azure、本地 sherpa 语音等路径，直播回复可播报。
- 语音陪伴支持 VAD、ASR、TTS、麦克风增益、回复长度、温度和语音性别配置。
- 实时变声器支持 RVC 模型搜索、下载、切换、启动、停止和 PTH 转 ONNX。

### 3.5 观众档案与统计

- 本地记录互动事件，保留规范化字段和原始 JSON。
- 自动维护用户画像统计，支持手动添加、恢复、软删除、备注、别名和用户专属 TTS 音色。
- 数据统计页展示弹幕、进场、关注、分享、礼物、舰长、人气、未知事件、盲盒盈亏、每日趋势和礼物排行。

### 3.6 插件中心

- 弹幕聊天：OBS 浏览器源，支持主题、字体、颜色、头像、徽章、礼物、SC、动画和自定义 CSS。
- 音乐互动：礼物/SC 产生点歌权益，弹幕搜索和确认歌曲，管理队列，生成 OBS 点歌页面。
- 心愿目标：按礼物累计目标，支持样式、音效、模拟和重置。
- 抽奖互动：按礼物门槛和停留时间抽奖，维护奖品列表。
- 礼物特效：按礼物触发直播画面特效和音效。
- 最近礼物：展示最近礼物列表。
- 礼物排行：展示贡献排行。

## 4. 当前进度

- 桌面端主流程已形成：登录、连接房间、启动监听、发送弹幕、配置保存、插件 HTTP 服务、自动更新。
- 直播事件记录和统计已落地到 SQLite，并用于观众档案和数据统计。
- 插件中心已从单一弹幕聊天扩展到音乐互动、心愿目标、抽奖、礼物特效、最近礼物和礼物排行。
- 音乐互动已经具备点歌权益、搜索、确认、队列、播放状态流转和 OBS 页面。
- 语音链路已包含本地模型下载、TTS 预览、VAD/ASR 配置、TTS 播报和 RVC 变声器。

## 5. 非目标

- 当前不把 `docs/superpowers` 作为产品文档来源整理，它保留为研发过程资料。
- 当前不维护 Slint 桌面版设计，实际实现以 Tauri + React 前端为准。
- QQ 音乐等非网易云搜索源尚未完成 Provider 接入，相关 UI 只能作为选项/约束展示。
