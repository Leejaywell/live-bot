# 技术路线图 (Roadmap)

本项目是一款专注于 Bilibili 的直播间智能场控与互动助手（Streamix）。随着桌面端全新 Tauri 架构的落地，我们的演进方向正式从“单一事件回复工具”转向“具备全景感知与决策能力的 AI Agent”。

## 当前方向

下一阶段的核心是**夯实本地数据循环**与**打通 AI 场控的感知链路**。在推进多平台支持前，机器人首先需要在单房间内做到：无遗漏地记录全类互动事实、提供直观的数据图表反馈，并能够通过大语言模型与语音模块实现拟人化的互动。

## 已确认的架构决策

- **UI 框架体系**：全面采用 Tauri 2.0 (React + TypeScript) 替代原有的 Slint/纯终端方案，以支持更复杂的数据可视化（F6 统计图表）与复杂的 AI 配置面板（F5 场控）。
- **数据保留原则**：互动记录 (Interaction Records) 在落库时，必须同时保留规范化字段和**原始 Bilibili JSON 负载 (Payload)**，以便未来重放数据或适配 B 站接口变更。
- **场次生命周期**：“直播场次 (Live Session)”严格遵循 Bilibili 房间的开播/下播状态，不与本软件的进程重启或重连绑定。
- **非阻塞存储**：所有 SQLite 的写入通过队列或锁机制处理，写入失败记录日志但**绝不阻塞**现有的弹幕交互引擎。
- **配置与数据分离**：用户配置存放于 `etc/` 的 YAML，身份凭证存放于 `token/`，海量事实数据存入 `db/`。

## 里程碑规划

### V1: 数据循环与现代化基座 (当前阶段)

**目标**：构建稳固的 SQLite 持久化层，并完成具有 8 大核心模块的桌面端界面。

- **实施范围**：
  - [x] 完成 Tauri 前端框架的重构，落地 8 大导航模块（仪表盘、登录、监听、回复、AI 场控、统计、PK、系统）。
  - [x] 在 `crates/bilibili-live-protocol` 引入 `ParsedLiveEvent` 包装器保留原始 JSON。
  - [x] 实现 `live_sessions` 与 `interaction_records` 的基础写入闭环。
  - [x] 实现支持本场、今日、近7天的数据聚合查询（弹幕数、进场、礼物价值、PK 胜负）。
  - [ ] 在前端完善 F6 数据统计面板的趋势图与环形图渲染。

### V2: AI 场控与感官唤醒 (Advanced AI Agent)

**目标**：在 F5 “AI 场控” 面板中，让机器人拥有思考的大脑与听/说的感官。

- **逻辑层 (Logic)**：
  - 接入 OpenAI 兼容 API、DeepSeek 等云端模型。
  - 支持连接本地 Ollama/LocalAI，实现零 Token 成本的隐私对话。
  - 注入直播间实时状态（如当前人气、PK 得分）作为 AI 的上下文 (System Prompt)。
- **感官层 (Voice)**：
  - **听觉 (ASR)**：集成 Faster-Whisper 或 FunASR，实现麦克风语音指令的精准识别。
  - **视觉 (TTS)**：集成 Bert-VITS2 或 ChatTTS，让 AI 用定制音色播报弹幕或回复。
- **情感与氛围决策**：AI 分析弹幕情绪波动，在检测到负面刷屏或异常事件时，向主播输出安全预警。

### V3: 记忆与生态自动化 (Memory & Ecosystem)

**目标**：让 AI 从“单次对话”升级为“拥有记忆的场控助手”。

- **长期记忆 (Memory)**：
  - 引入轻量级向量数据库 (如 Qdrant 或本地嵌入)。
  - 实现基于 RAG 的“主播百科”，AI 能够准确回答直播时间、外设配置等 FAQ。
  - 建立观众亲密度档案，AI 能认出老粉并提供个性化欢迎。
- **主动运营体系**：
  - AI 摆脱被动回复，根据热度主动提议：“当前在线 5000 人，是否开启天选抽奖？”
  - 定时广播与 Webhook 推送联动（如高额打赏自动发至粉丝群）。

### V4: 跨平台协议抽象 (Platform Abstraction Layer)

**目标**：突破 Bilibili 单一平台限制，演进为通用直播助手。

- **协议标准化**：定义全局 `GenericLiveEvent` 类型，将各平台私有协议映射为通用格式。
- **插件化接入**：通过 Provider 模式，低侵入式地接入抖音 (TikTok)、虎牙、斗鱼等平台 WebSocket。
- **多平台数据融合**：UI 重构为多房间并发视图，支持跨平台的弹幕聚合与统计展示。

### V5: 终极智能体 (Autonomous AI Assistant)

**目标**：实现 AI 对直播环境的物理级感知与操作 (Computer Use)。

- **场景化系统控制**：
  - **直播软件联动**：通过 WebSocket 控制 OBS 进行场景切换、音频调整和特定源的隐现。
  - **媒体控制**：AI 接收观众点歌指令，自动操作网易云/QQ音乐切歌。
- **安全沙箱 (Sandboxing)**：
  - 建立严格的 Function Calling 指令白名单，所有系统级操作（如键鼠模拟、进程启动）必须经由预授权。
