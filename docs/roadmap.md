# 流光 Roadmap

更新日期：2026-05-20

## 1. 当前状态

已完成并可继续打磨的主线：

- Tauri 桌面端主应用。
- Bilibili 登录、房间连接、监听和发送弹幕。
- 直播事件记录、用户画像和数据统计。
- 自动欢迎、关注/分享答谢、礼物/SC 答谢、定时弹幕、过滤规则。
- AI Provider、AI Bot、AgentRuntime、会话记忆和基础工具调用。
- TTS 播报、实时弹幕朗读、语音陪伴配置、本地模型管理。
- OBS 插件中心：弹幕聊天、音乐互动、心愿目标、抽奖、礼物特效、最近礼物、礼物排行。
- 音乐互动：权益、搜索、确认、队列、播放状态和 OBS 展示。

## 2. 近期优先级

### P0：稳定当前直播主链路

- 梳理 monitor 事件处理顺序，补充关键链路测试。
- 保证启动监听、停止监听、重载 TTS、重载语音配置、房间恢复的状态一致。
- 明确 AI 页面测试路径与直播 Agent 路径差异，避免用户误判。
- 为配置保存失败、端口占用、登录失效、模型缺失提供更明确 UI 提示。

### P1：补齐音乐互动闭环

- 接入更多音乐 Provider，优先处理 QQ 音乐或明确隐藏未接入选项。
- 完善 `换一批`、取消点歌、重复点歌惩罚和黑名单 UI。
- 补齐自动下一首与手动确认两种播放模式的端到端验证。
- 点歌 OBS 页面增加空状态、播放失败和队列变化动效。

### P1：完善插件中心

- 统一各插件的 URL 复制、模拟、重置、保存反馈和错误提示样式。
- 为礼物目录刷新失败增加降级提示。
- 插件配置需要导入/导出能力，方便主播迁移。

### P2：语音体验提升

- 把 VAD、ASR、TTS 的运行状态和错误拆成更清晰的诊断面板。
- 增强本地模型下载失败重试和镜像选择。
- 评估语音陪伴与弹幕朗读同时开启时的优先级和互斥策略。
- 继续打磨 RVC 模型下载、转换和运行状态提示。

### P2：发布与运维

- 稳定 macOS/Windows 打包和自动更新。
- 为公开发布仓库补齐安装说明、更新说明和问题排查。
- 增加日志导出，方便定位用户现场问题。

## 3. 中期方向

- 场控策略模板：按直播类型导入整套欢迎、答谢、AI 和插件配置。
- 观众分层运营：基于礼物、活跃度、互动类型自动分组。
- AI 工具扩展：查询观众画像、控制插件状态、执行 OBS 动作。
- 更完整的直播数据报表：场次对比、留存、付费、互动质量。
- 插件市场化：插件预设、皮肤、音效和主题包。

## 4. 暂缓事项

- Slint 桌面版本：当前实现已转向 Tauri + React。
- 完整本地大模型对话：当前优先使用 OpenAI-compatible 云端 Provider。
- 大规模云同步：当前定位仍是本地桌面工具，数据以本地 SQLite 为主。
