## 四、弹幕朗读特殊考量

弹幕场景和语音对话不同，需要额外处理：

| 问题 | 方案 |
|------|------|
| 弹幕爆发时队列积压 | 超时丢弃旧弹幕，只念最新 N 条 |
| 礼物 / SC 优先级高于普通弹幕 | `SpeakerRouter` 优先级队列 ✅ 已实现 |
| 敏感词 / 特殊字符 | 前置过滤，emoji 转描述或静音 |
| 多音字 / 数字读法 | 文本规范化（TN），讯飞/阿里云内置，本地需额外处理 |
| 低延迟要求 | 本地 MeloTTS 或 Edge TTS WebSocket 流式 |
| 主播名字念错 | 自定义词典 / 热词（讯飞/阿里云支持） |

## 六、场景方案

### 场景一：情感疗愈　`需要 LLM`

**核心诉求**：声音治愈、情绪感知、陪伴感强、不能有机械感。

#### 技术栈

| 层级 | 方案 | 理由 |
|------|------|------|
| **TTS** | ChatTTS（本地）或 ElevenLabs（云）| ChatTTS 支持 `[laugh]` `[uv_break]` 语气词，天然有温度；ElevenLabs 情感克隆质量最高 |
| **TTS 备选** | CosyVoice 2 + 情感标签 | `<\|HAPPY\|>` `<\|SAD\|>` 控制语气，可克隆主播声音 |
| **ASR** | SenseVoice（本地）| 输出情绪标签：`<HAPPY>` `<FEARFUL>` `<DISGUSTED>`，透传给 AI |
| **AI** | Claude / GPT-4o + 疗愈人设 | system prompt 设定共情风格，接收情绪标签作上下文 |
| **背景音** | 白噪音 / 雨声 / 频率音叉 | 通过 `SpeakerRouter` 最低优先级通道持续播放 |
| **VAD** | Silero | 检测主播说话 → 背景音淡出；主播停说 → 背景音淡入 |

#### 数据流

```
主播语音
  → Silero VAD（检测话段）
  → SenseVoice ASR（文字 + 情绪标签）
  → AI（带情绪上下文 → 共情回复）
  → ChatTTS（温柔语气合成）
  → 播放（背景音淡出 → TTS → 背景音淡入）
```

#### 关键实现点

- SenseVoice 情绪标签透传 AI prompt：`用户语气：[难过]，请用温柔的方式回应`
- ChatTTS 文本预处理：自动插入停顿符，语句不急促
- 弹幕朗读用最柔和的声线，过滤攻击性内容

### 场景二：聊天声控　`需要 LLM（意图识别兜底）`

**核心诉求**：低延迟、指令准确、能控制 OBS / 弹幕 / 机器人，语音版遥控器。

#### 技术栈

| 层级 | 方案 | 理由 |
|------|------|------|
| **VAD** | Silero | 即时检测说话开始，不能有明显延迟 |
| **ASR** | Moonshine（本地）或 Faster-Whisper tiny/base | Moonshine 专为实时设计，首字延迟 < 300ms；tiny 模型 CPU 可跑 |
| **指令解析** | 关键词匹配 → LLM 兜底 | 高频指令本地关键词；模糊指令走 LLM |
| **TTS 反馈** | MeloTTS（本地）或 Edge TTS | 短句确认反馈，MeloTTS 实时因子 > 10x |
| **控制总线** | Tauri 命令 + OBS WebSocket | 识别到指令直接 invoke，OBS 场景切换走 `obs_enabled` 通道 |

#### 指令分层

```
"切游戏场景"       → 本地关键词 → OBS 切 scene（< 500ms）
"帮我发个弹幕XXX"  → 本地关键词 → 发送弹幕
"开始摸鱼模式"     → LLM 意图识别 → 组合动作（切场景 + 停止 AI + 放音乐）
"让二狗说个笑话"   → LLM → 触发指定 bot
```

#### 数据流

```
主播语音
  → VAD（检测话段结束）
  → ASR（快速转文字）
  → 关键词树（命中 → 立即执行）
      ↓ 未命中
  → LLM 意图识别（异步，< 1s）
      → 执行动作 + TTS 短句确认
```

#### 关键实现点

- 静音阈值调短（0.3s），减少等待感
- 指令执行失败要有 TTS 反馈，主播知道有没有听到
- 热词列表做成配置项，主播自定义

### 场景三：整活变声器　`不需要 LLM`

**核心诉求**：实时变声、音效丰富、低延迟（< 50ms）、有趣。

#### 技术栈

| 层级 | 方案 | 理由 |
|------|------|------|
| **实时变声（音色克隆）** | RVC v2 + w-okada | 行业标准，实时运行，声库社区庞大（猫娘/萝莉/大叔/鬼畜）|
| **变声引擎 UI** | Applio（RVC fork）| 现代化界面，一键导入声库，支持 ONNX 推理 |
| **音调变换** | RubberBand | 不改音色只变音调：chipmunk (+12)、巨人 (-12)、变速鬼畜 |
| **特效** | SoX / LADSPA 插件 | 机器人（ring mod）、电话音（bandpass）、回声、混响 |
| **虚拟声卡** | BlackHole（macOS）/ VB-Audio（Windows）| 变声输出路由到 OBS 麦克风输入 |
| **TTS 整活** | BERT-VITS2 | 输入任意文字，用名人/角色音色念出来，鬼畜素材 |

#### 变声模式菜单

| 模式 | 技术实现 |
|------|---------|
| 萝莉 / 御姐 | RVC 声库切换 |
| 机器人 | ring modulator（正弦波调制）+ 混响 |
| 松鼠 / 花栗鼠 | pitch shift +8~12 半音 + 加速 |
| 巨人 / 恶魔 | pitch shift -8~12 半音 + 混响 |
| 电话 | 300~3400 Hz bandpass + 白噪底噪 |
| 合唱 | 延迟叠加（3-5 路，微偏音调）|
| 鬼畜节奏 | 实时切片 + 节拍重复（BPM 对齐）|

#### 数据流

```
麦克风输入
  → 前处理（降噪 / 去混响）
  → 变声引擎（RVC 实时推理 / pitch shift）
  → 音效链（ring mod / reverb / eq）
  → 虚拟声卡输出 → OBS 采集
         ↓
  同时送入 ASR → 弹幕互动 / 声控指令
  （识别原始声音，不识别变声后）
```

#### 集成策略

变声器建议作**独立进程**运行（RVC / w-okada），Streamix 通过 WebSocket 或虚拟音频设备对接，不把实时音频处理管线内置——变声对实时性要求极高，单独进程更好调优。

Streamix 负责：
- 弹幕触发变声切换（收到"切机器人" → 切模式）
- 声控指令识别（走原始麦克风路，不走变声链路）
- 变声状态显示在 UI

## 七、三场景横向对比

| | 情感疗愈 | 聊天声控 | 整活变声器 |
|--|---------|---------|----------|
| **需要 LLM** | ✅ 是（云端）| ✅ 是（云端，仅兜底）| ❌ 否 |
| **延迟要求** | 低（2-5s 可接受）| 极低（< 1s）| 极低（< 50ms）|
| **LLM 推荐** | ☁️ Claude / Qwen-Max / DeepSeek | ☁️ DeepSeek / Doubao | — |
| **TTS** | Kokoro ONNX（本地）/ Fish Speech（本地，克隆）| Kokoro ONNX（本地）| BERT-VITS2 |
| **ASR** | SenseVoice / Whisper（本地）| Paraformer / SenseVoice（本地）| 原始麦克风路 |
| **核心组件** | 情绪感知 + 云端 AI 共情 | 本地指令树 + 云端兜底 | RVC + 音效链 |
| **硬件要求** | 低（TTS/ASR 本地，< 500MB）| 极低（< 400MB）| 中高（GPU 推荐）|
| **Streamix 内置** | 可完整内置 | 可完整内置 | 建议独立进程 |
