# 流光 UI 设计文档

更新日期：2026-05-20

## 1. 当前 UI 基线

当前 UI 是 Tauri + React 桌面应用，不再以 Slint/Figma 提示稿为准。主品牌名在界面中显示为「流光」。

整体风格是玻璃拟态工作台：侧栏、顶栏、卡片、弹窗、主题面板和通知面板使用半透明背景、模糊、阴影和主题色。前端同时支持浅色/暗色、主题色、背景效果、光标效果和点击涟漪。

## 2. 信息架构

主侧栏模块：

- 直播概览
- 观众档案
- 数据统计
- 触发回复
- 实时弹幕
- 智能助手
- 语音陪伴
- 模型管理
- 插件中心

插件中心子模块：

- 弹幕聊天
- 音乐互动
- 心愿目标
- 抽奖互动
- 礼物特效
- 最近礼物
- 礼物排行

## 3. 全局布局

- 顶栏：展示登录/房间状态、主题、通知、设置等入口。
- 左侧栏：支持展开、图标和隐藏三种状态。
- 主内容区：按页面组织业务卡片和操作区。
- 设置面板：管理全局体验类选项，例如关闭背景效果、关闭光标效果、配置目录等。
- 通知面板：由监听日志转化关注、礼物、SC、错误等通知。

## 4. 页面设计

### 4.1 直播概览

定位为运营总览页。

- 展示监听状态、本场统计和自动化模块状态。
- 自动化模块分组展示 AI、欢迎语、礼物/播报、活动通知、数据记录、语音陪伴、定时弹幕、弹幕过滤等。
- 快速入口跳转到对应配置页。

### 4.2 观众档案

定位为用户 CRM。

- 列表展示 UID、昵称、别名、备注、最后出现、弹幕数、礼物价值等。
- 支持搜索、排序、添加、编辑、软删除和恢复。
- 支持为特定用户指定 TTS Provider 和音色。

### 4.3 数据统计

定位为复盘页。

- 支持周期筛选。
- 展示弹幕、进场、关注、分享、礼物、舰长、人气等汇总。
- 包含礼物排行、用户礼物排行、盲盒盈亏和趋势图。

### 4.4 触发回复

定位为自动化配置页。

- 使用 tab 分区：欢迎语、粉丝互动、礼物感谢、定时任务、过滤规则、系统项。
- 列表类配置应支持新增、删除、保存反馈。
- 高风险删除操作使用二次确认。

### 4.5 实时弹幕

定位为直播时工作页。

- 展示监听状态和实时事件流。
- 支持按事件类型过滤、暂停滚动、新消息提示。
- 支持手动发送弹幕。
- 支持弹幕朗读开关、TTS Provider、音色和语速配置。

### 4.6 智能助手

定位为 AI 配置与测试页。

- 管理 LLM/ASR/TTS Provider。
- 管理 AI Bot。
- 支持测试对话和 TTS 预览。
- 页面提示应区分测试对话和直播真实 Agent 路径。

### 4.7 语音陪伴

定位为主播语音输入和 AI 语音输出配置页。

- 配置 VAD、ASR、TTS、音色、语速、音调、麦克风增益、回复长度、温度。
- 管理语音陪伴开关和刷新运行时语音配置。
- 与模型管理联动提示本地模型是否已下载。

### 4.8 模型管理

定位为本地语音模型资产页。

- 检查 VAD、SenseVoice、Paraformer、Whisper、Kokoro、MeloTTS、Piper、RVC 等模型状态。
- 支持下载、取消、删除和打开模型目录。
- 进度展示需要区分下载、解压、完成和错误状态。

### 4.9 插件中心

定位为 OBS 画面管理页。

- 每个插件页都应提供 URL 复制、配置保存、预览或模拟能力。
- 需要明确区分「主控端配置」和「OBS 浏览器源展示」。
- 对音乐互动这类复杂插件，配置应分为开关、OBS 样式、播放方式、礼物档位、队列管理和测试搜索。

## 5. OBS 浏览器源设计

OBS 页面应满足：

- 默认透明背景或可配置背景。
- 字体、颜色、尺寸和动画可调。
- 页面尺寸可控，适应浏览器源裁切。
- 状态来自本地 HTTP 服务和 WebSocket，不依赖桌面 UI 是否停留在对应页面。

当前 OBS 插件包括弹幕聊天、音乐互动、心愿目标、抽奖互动、礼物特效、最近礼物和礼物排行。

## 6. 交互原则

- 直播中高频操作放在一级页面或快捷操作中。
- 配置保存需要即时反馈，失败必须保留用户输入。
- 需要房间连接的页面入口在未连接时打开房间弹窗。
- 删除、重置、退出程序等操作必须二次确认。
- 复制 URL、模拟事件、刷新礼物目录等操作要有明确 toast。
